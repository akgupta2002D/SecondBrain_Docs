question_extractor_ocr.urls

question_extractor_ocr.urls.urlpatterns

URL Patterns:

  1. upload/ - View: views.upload_view - Name: ‘upload_and_view’ - Description: Handles file uploads and renders the upload form.

  2. extract/ - View: views.extract_text_view - Name: ‘extract_text’ - Description: Processes text extraction from uploaded files.

Usage:

These URL patterns should be included in the main urls.py file of the project. Example:

from django.urls import include, path

urlpatterns = [

path(‘ocr/’, include(‘ocr_app.urls’)),

]

Note

Ensure that the view functions (upload_view and extract_text_view) are properly imported and defined in the views.py file of this application.

question_extractor_ocr.views

Views for handling file uploads and text extraction in the OCR application.

This module contains views for uploading files and extracting text using OCR. It interacts with the UploadedFile model and FileUploadForm.

question_extractor_ocr.views.upload_view(request)[source]

Handle file uploads and render the upload form.

This view processes POST requests for file uploads and renders the upload form for GET requests. It uses AJAX to handle file uploads asynchronously.

Parameters:

request (HttpRequest) – The request object.

Returns:

Rendered upload form for GET requests. JsonResponse: JSON response for POST requests, indicating success or failure.

question_extractor_ocr.views.extract_text_view(request)[source]

Extract text from an uploaded file.

This view processes POST requests to extract text from a previously uploaded file. It updates the UploadedFile instance with the extracted text.

Parameters:

request (HttpRequest) – The request object.

Returns:

JSON response indicating success or failure, including extracted text if successful.

class question_extractor_ocr.models.UploadedFile(*args, **kwargs)[source]

Bases: Model

Represents a file uploaded for OCR processing.

Stores information about uploaded files, including the file itself, upload timestamp, extracted text, and processing status.

Fields:

file: The uploaded file, stored in ‘uploads/’ directory. uploaded_at: Timestamp of file upload, auto-set on creation. extracted_text: Text extracted from the file (optional). is_processed: Boolean indicating if the file has been processed.
